# Value-conversion patterns

Generated by `excel-com-client-kb`; do not edit by hand. This is source-derived client-implementation evidence, not Excel runtime evidence unless explicitly labelled Control-confirmed.

Both clients implement client-specific mappings rather than exposing a raw public `VARIANT` model.

| VARTYPE | pywin32 result | comtypes result |
| --- | --- | --- |
| `VT_EMPTY`, `VT_NULL` | `None` | `None` |
| `VT_BOOL` | `bool` | `bool` |
| `VT_I2`, `VT_I4`, integral variants | Python `int` | Python `int` |
| `VT_R4`, `VT_R8` | `float` | `float` |
| `VT_DATE` | pywin date/time object | `datetime` |
| `VT_CY` | `Decimal` conversion | `Decimal` |
| `VT_BSTR` | `str` | `str` |
| `VT_ERROR` | integer scode | conversion is incomplete for some getter forms; error path is `COMError` |
| `VT_DISPATCH`, `VT_UNKNOWN` | COM wrapper then Dispatch selection | AddRef then dynamic/generated interface wrapper |

`Range.Value`, `Value2`, `Formula`, and `Formula2` are included as transport members; this table does not claim their Excel runtime result types.
