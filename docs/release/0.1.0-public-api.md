# `excel-com` 0.1.0 public API inventory

The deterministic machine-readable inventory is
[`0.1.0-public-api.txt`](0.1.0-public-api.txt), generated with
`cargo public-api -p excel-com -sss --all-features`. It is the complete list
of public modules, items, methods, constants, fields, and feature-visible API
for this first release baseline.

Every item in that snapshot is classified **retain**, except the explicit
pre-release corrections below. Since 0.1.0 is the first published API, names
are corrected now rather than deprecated immediately.

| Domain | Public path | Classification |
|---|---|---|
| Automation | `automation` and common values at root | retain |
| Sessions | root `ComApartment`, `OwnedApplication`, `AttachedApplication`, `ExcelSession` | retain |
| Application | root `Application` | retain; remove `Application::new` |
| Workbook / worksheet / range | root core types | retain |
| Formulas | root common types; `data` for related utilities | retain |
| Formatting | `formatting` | retain; canonical specialized module added |
| Tables | `tables` | retain; canonical specialized module added |
| Presentation | `presentation` | retain; canonical specialized module added |
| Drawing | `drawing` | retain; canonical specialized module added |
| Data utilities | `data` | retain; canonical specialized module added |
| External data | `external_data` | retain; canonical specialized module added |
| Pivots | `pivot` | retain; canonical specialized module added |
| Errors | root `ExcelComError`, `ExcelRuntimeError`, `InvocationError`; `error` | retain; make raw retry classification private |

Removed before release: `Application::new`, `ComMessageFilterGuard`,
`ComCallDisposition`, `InvocationRetrySafety`, and `XlSheetVisibility`.
`InvocationError` retains stable accessors for object type, member, DISPID,
HRESULT, EXCEPINFO, argument index, attempts, and elapsed time while keeping
the dispatch-retry implementation detail private.

The snapshot is the baseline for future `cargo public-api diff` and
`cargo-semver-checks` comparison after 0.1.0 is on crates.io.
