# `excel-com` 0.1.0 unsafe-code audit

The release audit found **62 `unsafe` blocks** and **22 unsafe function or
extern declarations** in `excel-com/src`. All unsafe operations are confined
to the private Automation and internal COM kernel; public wrappers neither
accept nor expose raw COM pointers, `VARIANT`, `SAFEARRAY`, BSTR pointers,
handles, PIDs, or HWNDs. Every block has a nearby `SAFETY:` justification and
the workspace denies `unsafe_op_in_unsafe_fn` and undocumented unsafe blocks.

| Area | Operations audited | Invariants, cleanup, and coverage |
|---|---|---|
| `internal/apartment.rs` | `CoInitializeEx`, `CoUninitialize` | `ComApartment` records the creating thread, is `!Send + !Sync`, and balances only successful STA initialization; Drop is non-panicking. Static and compile-fail tests cover affinity. |
| `internal/com_ptr.rs` | vtable dereference, QueryInterface, AddRef, Release | `ComPtr` owns exactly one non-null interface reference. QueryInterface and Clone establish one owned reference; Drop releases once. The type is private and `!Send + !Sync`. |
| `automation/dispatch.rs` | ProgID resolution, activation, Invoke, BSTR reads | Inputs and output storage remain valid through each ABI call. `DISPPARAMS` is private; EXCEPINFO BSTRs are copied and freed through owned variants. Invocation tests preserve HRESULT and redact addresses. |
| `automation/variant.rs` | VariantInit/Clear and tagged union access | Every `OwnedVariant` is initialized before mutation and cleared exactly once. Union reads are guarded by VARTYPE checks; dispatch transfer clears the source tag/pointer before ownership moves. Unit tests cover scalar, dispatch, error, date, currency, and BSTR paths. |
| `automation/bstr.rs` | `SysAllocStringLen`, `SysFreeString` | BSTR length, not NUL termination, defines contents. UTF-16 is copied without lossy conversion; Windows paths reject embedded NUL when termination is required. Owned allocation is freed once. |
| `automation/safearray.rs` and `value.rs` | SAFEARRAY creation, metadata, get/put, destroy | Array rank, VT_VARIANT element type, lower bounds, dimensions, and element counts are validated. Public arrays are zero-based row-major; Excel values are encoded with one-based SAFEARRAY bounds and decoded from actual bounds. Partial failure drops initialized owners. |
| `automation/enumerator.rs` | `IEnumVARIANT::Next` and vtable lookup | One owned enumerator is held by a fallible, fused iterator. Each yielded VARIANT is initialized, converted, and cleaned; early iterator drop releases the COM reference. |
| `automation/message_filter.rs` | IMessageFilter vtable and registration | A boxed filter stays alive until restored. COM callbacks validate null outputs, maintain non-owning IUnknown accounting, and never free the boxed allocation. Drop restores the prior filter best-effort and never panics. Machinery is private. |
| `excel/session.rs` | ROT lookup and process wait diagnostics | Attachment obtains and owns only the returned active-object reference. Owned-session diagnostics use an Excel window's process id, query/synchronize access only, close every handle, and never terminate a process. |

## VARIANT, SAFEARRAY, and BSTR conclusions

`VariantInit` precedes use and `VariantClear` runs once for every owned value.
BSTR ownership moves only into an owned variant and is never read as NUL-ended
text. Dispatch values receive an AddRef before entering a variant. SAFEARRAY
ownership is transferred exactly once between `SafeArray` and `OwnedVariant`;
borrowed reads never destroy the source. Rank-two rectangular `VT_ARRAY |
VT_VARIANT` values, non-zero lower bounds, empty/invalid shape rejection, and
nested-array rejection are covered by focused conversion tests. No inactive
union field is read.

## Panic, redaction, and Drop conclusions

All public failures return `ExcelComError`; restoration guards retain primary
operation failures and never panic in Drop. Password option debug output,
`SecretStringValue` Debug/Display, and connection/Power Query test coverage
confirm redaction. The audit intentionally records no machine paths, raw
pointer values, PIDs, or HWNDs in committed evidence.
